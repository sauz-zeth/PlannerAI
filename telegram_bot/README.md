# Telegram бот для тестирования AI Planner Backend API

Этот бот позволяет авторизоваться через Google Calendar и тестировать все API endpoints бэкенда.

## Установка

1. Установите зависимости:
```bash
cd telegram_bot
uv sync
```

2. Настройте переменные окружения в файле `.env` (в корне проекта):
```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
BACKEND_API_URL=http://localhost:8000
```

## Получение токена бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен в `.env`

## Настройка Deep Link

Для использования deep link авторизации:

1. В [@BotFather](https://t.me/BotFather) настройте команду `/setcommands`
2. Добавьте команды бота (опционально)

Deep link формат:
```
https://t.me/YOUR_BOT_USERNAME?start=AUTH_CODE
```

## Запуск

```bash
cd telegram_bot
./run.sh
```

Или напрямую:
```bash
cd telegram_bot
uv run python bot.py
```

## Команды бота

### Авторизация
- `/start` - Начать работу с ботом
- `/login` - Авторизоваться через Google Calendar
- `/check` - Проверить статус авторизации

### Работа с событиями
- `/events [количество]` - Получить события календаря
- `/upcoming [часы]` - Ближайшие события
- `/summary` - Статистика календаря
- `/status` - Статус подключения к календарю

### Создание и поиск
- `/create_event <название> <начало> <окончание> [описание]` - Создать событие
- `/search <текст>` - Поиск событий по тексту
- `/free_slots [дата] [длительность_мин]` - Найти свободные слоты

### Справка
- `/help` - Показать справку по командам

## Примеры использования

### Авторизация
1. Отправьте `/login` боту
2. Нажмите на кнопку "Авторизоваться через Google"
3. Завершите авторизацию в браузере
4. Вернитесь в бот и отправьте `/check`

### Получение событий
```
/events 5
```

### Создание события
```
/create_event Встреча 2024-01-20T14:00:00 2024-01-20T15:00:00 Описание встречи
```

### Поиск событий
```
/search встреча
```

### Поиск свободных слотов
```
/free_slots 2024-01-20 60
```

## Примечания

- Токены пользователей хранятся в памяти (в продакшене используйте БД)
- Убедитесь, что бэкенд запущен и доступен по адресу `BACKEND_API_URL`
- Для работы требуется авторизация через Google Calendar
